## TVBlockEntity

The `TVBlockEntity` class manages the state and behavior of television blocks, including cassette playback, video rendering, and enderman interaction.

### Class Definition

```java
package net.mehvahdjukaar.vista.common.tv;

public class TVBlockEntity extends ItemDisplayTile
```

**Location:** `net.mehvahdjukaar.vista.common.tv.TVBlockEntity`  
**Parent Class:** `ItemDisplayTile` (Moonlight Lib)

### Fields

<ParamField path="paused" type="boolean" private>
  Whether the video playback is currently paused.
</ParamField>

<ParamField path="videoPlaybackTicks" type="int" private>
  The current playback position in ticks (20 ticks = 1 second).
</ParamField>

<ParamField path="connectedTvsAmount" type="int" private>
  The width (in blocks) of the connected TV grid. Minimum value is 1.
</ParamField>

<ParamField path="videoSource" type="IVideoSource" private>
  Client-side video source for rendering. Set to `IVideoSource.EMPTY` when no cassette is inserted.
</ParamField>

<ParamField path="observationController" type="TVEndermanObservationController" private>
  Server-side controller for enderman observation logic. Null if not broadcasting a viewfinder feed.
</ParamField>

<ParamField path="fadeAnimation" type="IntAnimationState" public>
  Animation controller for the screen fade-in/fade-out effect.
  
  Duration: 3 ticks fade-in, 9 ticks fade-out
</ParamField>

<ParamField path="endermanAnimation" type="IntAnimationState" public>
  Animation controller for the enderman static effect when looking at endermen through the TV.
  
  Duration: 20 ticks fade-in, 20 ticks fade-out, 0.6 intensity
</ParamField>

### Constructor

```java
public TVBlockEntity(BlockPos pos, BlockState state)
```

Creates a new TV block entity at the specified position.

### Key Methods

#### onTick (Static Ticker)

```java
public static void onTick(
    Level world, 
    BlockPos pos, 
    BlockState state, 
    TVBlockEntity tv
)
```

The main tick method called every game tick. Handles:
- Video playback progression
- Animation updates (fade, enderman static)
- Sound playback loops
- Enderman observation checks (server-side)

<ParamField path="world" type="Level">
  The world/level containing the TV.
</ParamField>

<ParamField path="pos" type="BlockPos">
  The TV's block position.
</ParamField>

<ParamField path="state" type="BlockState">
  The TV's current block state.
</ParamField>

<ParamField path="tv" type="TVBlockEntity">
  The TV block entity instance.
</ParamField>

<Expandable title="Tick Logic Details">
  **Client-side behavior:**
  - Increments fade animation when powered
  - Plays video sound in a loop based on video duration
  - Updates enderman static animation
  - Resets playback when unpowered
  
  **Server-side behavior:**
  - Staggered updates (every 27 ticks) for performance
  - Checks if endermen are visible through viewfinder feeds
  - Sends block events to sync enderman state to clients
  - Manages the observation controller lifecycle
</Expandable>

#### interactWithPlayerItem

```java
public ItemInteractionResult interactWithPlayerItem(
    Player player, 
    InteractionHand handIn, 
    ItemStack stack, 
    int slot, 
    BlockHitResult hit
)
```

Handles all player interactions with the TV:

1. **Pause Toggle**: Shift-right-click while powered with a cassette to pause/unpause
2. **Cassette Ejection**: Right-click with empty hand or another cassette to eject current cassette
3. **Cassette Insertion**: Right-click with a valid cassette item to insert it
4. **Advancement**: Triggers "Absolute Cinema" advancement when inserting into 3x3+ TV

<ParamField path="player" type="Player">
  The player interacting with the TV.
</ParamField>

<ParamField path="handIn" type="InteractionHand">
  The hand used for interaction.
</ParamField>

<ParamField path="stack" type="ItemStack">
  The item in the player's hand.
</ParamField>

<ParamField path="hit" type="BlockHitResult">
  The ray trace result for the interaction.
</ParamField>

<ResponseField name="Returns" type="ItemInteractionResult">
  - SUCCESS: Interaction handled
  - PASS_TO_DEFAULT_BLOCK_INTERACTION: Fall through to default behavior
</ResponseField>

### Example Usage

```java
import net.mehvahdjukaar.vista.common.tv.TVBlockEntity;
import net.minecraft.core.BlockPos;
import net.minecraft.world.level.Level;

public class TVController {
    public void pauseTV(Level level, BlockPos pos) {
        if (level.getBlockEntity(pos) instanceof TVBlockEntity tv) {
            // Pause/unpause the TV
            boolean wasPaused = tv.isPaused();
            // Note: Pausing is typically done through player interaction
            // Direct field access requires reflection or custom methods
        }
    }
    
    public void getPlaybackInfo(Level level, BlockPos pos) {
        if (level.getBlockEntity(pos) instanceof TVBlockEntity tv) {
            int ticks = tv.getPlaybackTicks();
            int seconds = ticks / 20;
            int gridSize = tv.getConnectedCount();
            boolean paused = tv.isPaused();
            
            System.out.println("Playback: " + seconds + "s, Grid: " + 
                gridSize + "x" + gridSize + ", Paused: " + paused);
        }
    }
}
```

## ViewFinderBlockEntity

The `ViewFinderBlockEntity` class manages viewfinder camera blocks, including position/orientation, zoom level, and video broadcasting.

### Class Definition

```java
package net.mehvahdjukaar.vista.common.view_finder;

public class ViewFinderBlockEntity extends ItemDisplayTile 
    implements IOneUserInteractable, IBroadcastSource
```

**Location:** `net.mehvahdjukaar.vista.common.view_finder.ViewFinderBlockEntity`  
**Parent Class:** `ItemDisplayTile` (Moonlight Lib)  
**Interfaces:** `IOneUserInteractable`, `IBroadcastSource`

### Fields

<ParamField path="myUUID" type="UUID" private>
  Unique identifier for this viewfinder's broadcast feed.
</ParamField>

<ParamField path="pitch" type="float" private>
  Current camera pitch in degrees (-90 to 90, where -90 is up, 90 is down).
</ParamField>

<ParamField path="yaw" type="float" private>
  Current camera yaw in degrees (0-360, where 0 is north).
</ParamField>

<ParamField path="prevPitch" type="float" private>

<ParamField path="prevYaw" type="float" private>

<ParamField path="zoom" type="int" private>

<ParamField path="locked" type="boolean" private>

</ParamField>

return Mth.lerp(normalizedZoom, 1, maxZoom);

### Broadcast Registration

The viewfinder automatically registers itself with the BroadcastManager:

```java
@Override
public void setLevel(Level level) {
    super.setLevel(level);
    this.ensureLinked(level, LevelBELocation.of(this));
}
```

<Warning>
  Always ensure viewfinders are properly unlinked when removed to prevent memory leaks in the broadcast system.
</Warning>

### Example Usage

```java
import net.mehvahdjukaar.vista.common.view_finder.ViewFinderBlockEntity;
import net.minecraft.core.BlockPos;
import net.minecraft.world.level.Level;

public class ViewFinderController {
    
    public void rotateViewFinder(Level level, BlockPos pos, float deltaYaw, float deltaPitch) {
        if (level.getBlockEntity(pos) instanceof ViewFinderBlockEntity vf) {
            float newYaw = vf.getYaw() + deltaYaw;
            float newPitch = vf.getPitch() + deltaPitch;
            
            vf.setYaw(vf.selfAccess, newYaw);
            vf.setPitch(vf.selfAccess, newPitch);
            vf.setChanged();
        }
    }
    
    public void setMaxZoom(Level level, BlockPos pos) {
        if (level.getBlockEntity(pos) instanceof ViewFinderBlockEntity vf) {
            vf.setZoomLevel(ViewFinderBlockEntity.MAX_ZOOM);
            vf.setChanged();
        }
    }
    
    public UUID getBroadcastId(Level level, BlockPos pos) {
        if (level.getBlockEntity(pos) instanceof ViewFinderBlockEntity vf) {
            return vf.getUUID();
        }
        return null;
    }
}
```
